# 사내강사용 PPT 자동 생성 도구 — 개발 계획

## 프로젝트 구조

```
ppt-generator/
├── main.py                  # CLI 진입점 + 오케스트레이션
├── parser.py                # Word 양식 파싱 ([키워드: 값] 포맷)
├── generator.py             # PPT 슬라이드 생성 (python-pptx)
├── config.py                # 슬라이드 좌표/폰트/스타일 상수 정의
├── requirements.txt         # python-docx, python-pptx
├── setup.bat                # Windows 설치 스크립트
├── run.bat                  # 실행 스크립트
├── template/
│   └── base_template.pptx   # 회사 표준 PPT (마스터 스타일 기준)
├── sample/
│   └── sample_input.docx    # 강사용 Word 양식 예시
└── README.md                # 설치 및 사용 가이드
```

## 샘플 분석 기반 슬라이드 타입 정의

샘플 PPT에서 발견된 **실제 사용 패턴** 3가지를 슬라이드 타입으로 정의합니다.

### 타입 1: 커버 (`1_제목 슬라이드` 레이아웃)
- **용도**: 강의 첫 슬라이드 (Slide 1)
- **구성**: 중앙 WordArt 타이틀 (`HY헤드라인M`, 24pt, 중앙정렬) + 회사 로고 이미지
- **Word 입력**: `[슬라이드 타입]: 커버`, `[제목]: ...`, `[부제목]: ...`

### 타입 2: 섹션헤더 (`1_제목 슬라이드` 레이아웃)
- **용도**: 각 챕터 시작 (Slide 3, 23, 27, 32, 49, 58)
- **구성**: 로마자 번호 + 섹션 제목 (`HY헤드라인M`, 24pt, 중앙정렬)
- **Word 입력**: `[슬라이드 타입]: 섹션헤더`, `[제목]: ...`
- **자동화**: 로마자 번호(I, II, III...) 자동 부여

### 타입 3: 본문 (`빈 화면` 레이아웃) — 핵심
- **용도**: 대부분의 콘텐츠 슬라이드 (55장/79장 = 70%)
- **공통 구성 요소** (69회 반복 패턴):
  ```
  ┌─────────────────────────────────────────────┐
  │ [섹션 표시 바]  상단 좌측                      │
  │   맑은 고딕 16pt, "Ⅰ. 섹션 제목"              │
  │                                              │
  │ [소제목]  좌측 상단                             │
  │   HY헤드라인M 20pt, "1. 정보의 가치 (1/5)"    │
  │                                              │
  │ ┌────────────────────────────────────────┐   │
  │ │ [본문 콘텐츠 영역]                       │   │
  │ │   맑은 고딕 14~22pt                     │   │
  │ │   - 불릿 포인트                          │   │
  │ │   - 불릿 포인트                          │   │
  │ └────────────────────────────────────────┘   │
  └─────────────────────────────────────────────┘
  ```
- **Word 입력**: `[슬라이드 타입]: 본문`, `[소제목]: ...`, `[내용]: ...`
- **섹션 표시 바**: 현재 속한 섹션의 제목을 자동으로 표시

### 타입 4: 목차 (`빈 화면` 레이아웃)
- **용도**: Slide 2 — 강의 전체 구조 보여줌
- **구성**: "< 목 차 >" 타이틀 + 섹션 제목 리스트 (맑은 고딕 28pt)
- **자동화**: 섹션헤더 슬라이드들의 제목을 자동 수집하여 생성
- **Word 입력**: 별도 입력 불필요 (자동 생성)

## 단계별 개발 계획

### Phase 1: 기반 구축 (parser.py + config.py)

**1-1. Word 양식 파서 (`parser.py`)**
- `[키워드]: 값` 형식의 .docx 파일을 파싱
- 슬라이드 단위로 구조화된 딕셔너리 리스트 반환
- 지원 키워드: `슬라이드 타입`, `제목`, `부제목`, `소제목`, `내용`
- `[내용]:` 아래의 불릿(`-`) 리스트를 배열로 수집

```python
# parser.py 출력 예시
[
    {"type": "커버", "제목": "정보보안경영시스템의 이해", "부제목": "HR 교육팀"},
    {"type": "섹션헤더", "제목": "정보자산의 가치 및 정보보호 요소"},
    {"type": "본문", "소제목": "1. 정보의 가치 (1/3)", "내용": ["정보는 비즈니스 자산", "적절한 보호 필요"]},
    {"type": "본문", "소제목": "1. 정보의 가치 (2/3)", "내용": ["기밀성", "무결성", "가용성"]},
    {"type": "섹션헤더", "제목": "정보보안경영시스템 개요"},
    ...
]
```

**1-2. 좌표/스타일 상수 (`config.py`)**
- 샘플에서 추출한 정확한 좌표값(EMU)과 폰트 설정 정의
- 슬라이드 크기: 10.83 × 7.50 inches (와이드스크린)
- 각 슬라이드 타입별 shape 위치, 크기, 폰트 상수

```python
# 샘플에서 추출한 실제 좌표
SECTION_BAR = {"left": 50800, "top": -47625, "width": ..., "height": ...}
SUBTITLE_BOX = {"left": 300038, "top": 388938, "width": ..., "height": ...}
CONTENT_AREA = {"left": ..., "top": ..., "width": ..., "height": ...}

FONT_TITLE = {"name": "HY헤드라인M", "size": Pt(24), "bold": False}
FONT_SUBTITLE = {"name": "HY헤드라인M", "size": Pt(20), "bold": False}
FONT_BODY = {"name": "맑은 고딕", "size": Pt(18), "bold": False}
FONT_SECTION_BAR = {"name": "맑은 고딕", "size": Pt(16), "bold": False}
```

### Phase 2: PPT 생성 엔진 (generator.py)

**2-1. 템플릿 기반 생성**
- `base_template.pptx`에서 마스터 슬라이드 스타일(배경, 색상 팔레트) 상속
- `빈 화면` 레이아웃 기반으로 슬라이드 추가 후 shape 수동 배치
- `1_제목 슬라이드` 레이아웃은 커버/섹션헤더용

**2-2. 슬라이드 타입별 생성 함수**
```python
def create_cover_slide(prs, title, subtitle)       # 커버
def create_section_slide(prs, title, section_num)   # 섹션헤더 (로마자 자동)
def create_content_slide(prs, subtitle, bullets, section_title)  # 본문
def create_toc_slide(prs, section_titles)            # 목차 (자동 생성)
```

**2-3. 본문 슬라이드 핵심 로직**
1. `빈 화면` 레이아웃으로 슬라이드 추가
2. 섹션 표시 바(`Rectangle 3`) 생성 — 현재 섹션 제목 표시
3. 소제목(`Text Box 68`) 생성 — `HY헤드라인M` 20pt
4. 콘텐츠 텍스트박스 생성 — `맑은 고딕` 불릿 포인트
5. 불릿 개수에 따라 폰트 크기 자동 조절 (5개 이하: 18pt, 10개 이하: 16pt, 이상: 14pt)

### Phase 3: CLI 및 통합 (main.py)

**3-1. CLI 인터페이스**
```bash
python main.py input.docx                    # 기본 (output.pptx 생성)
python main.py input.docx -o 강의안.pptx      # 출력 파일명 지정
python main.py input.docx -t custom.pptx     # 커스텀 템플릿 지정
```

**3-2. 파이프라인**
```
input.docx → parser.py → [슬라이드 딕셔너리 리스트]
                              ↓
                         generator.py → output.pptx
                              ↑
                    config.py (좌표/폰트 상수)
                    template/base_template.pptx (마스터 스타일)
```

### Phase 4: Word 양식 템플릿 + 배포

**4-1. sample_input.docx 생성**
- python-docx로 예시가 채워진 Word 양식 자동 생성
- 강사가 내용만 교체하면 되도록 구조 고정

**4-2. 배포 파일**
- `setup.bat`: Python 존재 확인 → venv 생성 → pip install → 바탕화면 바로가기
- `run.bat`: venv 활성화 → `python main.py %1` 실행 (파일 드래그&드롭 지원)
- `README.md`: 스크린샷 포함 설치/사용 가이드

## 기술적 고려사항

### 1. 템플릿 파일 준비
- 샘플 PPT에서 **빈 슬라이드만 남긴 템플릿** 추출 필요
- 슬라이드 마스터의 배경, 색상 테마, 로고 위치만 보존
- 방법: 샘플 .pptx를 열어 모든 슬라이드 삭제 후 저장 → `base_template.pptx`

### 2. HY헤드라인M 폰트
- 강사 PC에 해당 폰트가 설치되어 있어야 PPT에서 올바르게 표시됨
- 폰트가 없으면 대체 폰트로 렌더링 → 설치 세미나에서 확인 필요
- 대안: `맑은 고딕 Bold`로 폴백 옵션 제공

### 3. 본문 콘텐츠 자동 배치 한계
- MVP에서는 **텍스트(불릿 포인트)만** 자동 배치
- 이미지, 표, 다이어그램은 강사가 PPT 열어서 직접 추가
- 불릿 텍스트가 콘텐츠 영역을 넘치면 폰트 크기 자동 축소

### 4. 섹션 번호 자동화
- 섹션헤더 타입 슬라이드를 순서대로 만나면 I, II, III... 자동 부여
- 섹션 표시 바(Rectangle 3)의 텍스트에도 동일 번호 반영

## Word 양식 최종 포맷

```
[슬라이드 타입]: 커버
[제목]: 정보보안경영시스템의 이해와 위험관리
[부제목]: OO회사 보안팀

[슬라이드 타입]: 섹션헤더
[제목]: 정보자산의 가치 및 정보보호 요소

[슬라이드 타입]: 본문
[소제목]: 1. 정보의 가치 (1/3)
[내용]:
- 정보는 비즈니스 자산과 같이 가치를 가진 자산
- 따라서 적절하게 보호되어야 한다 (ISO 27001)
- 정보자산의 분류: 물리적/논리적/인적 자산

[슬라이드 타입]: 본문
[소제목]: 1. 정보의 가치 (2/3)
[내용]:
- 기밀성 (Confidentiality): 인가된 자만 접근
- 무결성 (Integrity): 정확성과 완전성 보장
- 가용성 (Availability): 필요 시 접근 가능

[슬라이드 타입]: 섹션헤더
[제목]: 정보보안경영시스템(ISO27001) 개요

[슬라이드 타입]: 본문
[소제목]: 1. ISO 27001 체계
[내용]:
- 14개 영역으로 구성
- 영역별 사규 제정 및 운영
- 보안책임자의 역할 정의
```

**목차 슬라이드**는 `[슬라이드 타입]: 섹션헤더`의 제목들을 자동 수집하여 커버 뒤에 자동 삽입됩니다.
